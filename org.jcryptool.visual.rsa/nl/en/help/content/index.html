RSA Cryptographic System

This help file consists of two parts, namely a Scheme Explanation and plug-in handling instructions.

Please note that in this visualization the operations are performed on each single character. In practice, the operations are performed on blocks or even the whole message. RSA is not used to encrypt large amount of data. Generally, it is used to encrypt a symmetric key, which encrypts the data using a symmetric cryptosystem, e.g., AES, and 3DES or to sign the hash value of a message.

1) Scheme Explanation

The most well-known asymmetric cryptosystem, the RSA algorithm, was developed in 1977 by Ronald Rivest, Adi Shamir and Leonard Adleman. The private and public keys are constructed using two big prime numbers p and q which the user can specify himself but must keep secret. The user chooses two natural numbers e and d such that (e*d) = 1 mod ((p-1)(q-1)).
The product of p*q = N and e then serves as the public key, N and d as the private key. For this encryption algorithm the plaintext letters must first be converted to numbers. This is generally done using the ASCII code, but one can of course also agree on A=0, B=1 .... Z=25. However, the number of each plaintext letter must be smaller than the product N. This becomes clear when one examines the encryption and decryption functions (a = plaintext letter, c = ciphertext letter): c = a^e mod N and a = c^d mod N
Since in the modulo division one can also first determine the remainder from each factor in the division modulo N and then the rest of its product modulo N, it would not always be possible to obtain a unique ciphertext character if a could be bigger than N.
The security of the RSA scheme is based on two mathematical problems. First, it is hard to extract a root modulo N (RSA-Problem). Second, factorization of a number N = p*q is hard, if the factors are sufficient large (RSA-Key problem).

a) RSA Key generation

A special characteristic of the RSA cryptosystem is that extensive calculations are necessary to generate the RSA key before RSA en-/decryption or signing can take place.
First of all the RSA parameters p, q, N and the Euler number phi(N) are calculated by choosing two different prime numbers p and q at random and then calculating the so-called RSA modulus N = p*q. The Euler number phi(N) = (p-1)*(q-1) is calculated from the prime factors p and q.
In a second step, the public RSA exponent e is determined and from this together with phi(N) the secret RSA exponent d is calculated. The number e is chosen as follows e: 1 < e < phi(N), with the property that e is relatively prime to phi(N). An especially popular value for e is 2^16+1 = 65537, as in most cases this is co-prime to phi(N) and is especially well-suited for square and multiply exponentiation resulting in a very fast public key operation.
The secret exponent d = e^(-1) mod phi(N) is calculated from the public exponent e as the multiplicative inverse modulo phi(N).
After the RSA key generation, the modulus N and the RSA key e can be published. Anyone could encrypt a message to the owner of the secret RSA key or check a digital signature.

b) RSA Encryption

To encrypt data with the RSA scheme, the ciphertext is calculated using the public key of the person who should be able to decrypt it, and the plaintext. The encryption function is c = a^e mod N. This means that anyone can encrypt a message using the public key. But only the owner of the associated secret key can then decrypt the message again.

To decrypt a ciphertext with the RSA scheme, one needs to know the private key d. The encryption function, c^d = (a^e)^d = a^(e*d) = a mod N, yields the plaintext a.

c) RSA Signature

To sign a message m with the RSA scheme, one needs to know the private key d. The signing function, m^d = sign(m) or hash(m)^d = sign(hash(m)), yields the signature.

One can verify a digital RSA signature using the signers public key e. Computing sign(m)^e = m' or sign(hash(m))^e = hash(m)', yields the result m' or hash(m)'. The result of the RSA exponentiation is m' or hash(m)' which has to be compared with the message m or the hash of the message hash(m) computed from the document. If both hash values are identical this is considered as a proof that the owner of the corresponding private RSA key has signed the document.

2) Plug-in Handling Instructions

When the RSA visualization is started, one can select the desired operation, encryption, decryption, sign, and verification, by choosing the corresponding tab. Then the key, which should be used, is required. Here, two options are given. First, create a new key or choose a saved key by clicking the "Key selection" button on the left. Second, create and store a new key by clicking the "Key generation..." button on the right. If you choose the second option, you have to load the created key by clicking the "Key selection" button. The chosen key is now automatically inserted in the can be seen in the "Key" field (converted to hex).
Next, click the "Enter text" button on the left. You can now enter a text using the characters a-z, A-Z, 0-9 and "SPACE". Every character will be converted to hex and can be seen in the "Entered text converted to numbers" field. If you mark the "Enter numbers directly" checkbox, you can insert decimal numbers separated by "SPACE". These numbers are converted to ASCII characters and can be seen in the "Entered text" field.
Now, all required information are given and it is possible to start the operation. If you just want to see the result, you can click on the "Calculate" button. To go stepwise through the calculation click on the "Start" button in the "Step-by-step calculations" field. If you go stepwise or directly through all steps, the result can be seen in the "Result" field.
On the bottom, you can see additional options. There, you have the possibility to inherit values from one of the three other tabs, show additional help while performing the operation, and reset the form.
